# English by Default

«بوابة إنجليزية» لأدوات الذكاء الاصطناعي التي تعمل على سطر الأوامر. تعترض الأداة أي طلب (prompt) غير مكتوب بالإنجليزية قبل أن يصل إلى النموذج، وتجبرك على إعادة صياغته بالإنجليزية بنفسك (أو الاستسلام لرؤية الترجمة الجاهزة والمتابعة)، ثم تحوّل كل زوج نصوص تجمعه - النص الأصلي مقابل صيغته الإنجليزية - إلى تمارين بأسلوب التكرار المتباعد (spaced repetition) وتقنية فاينمان (Feynman technique).

تعمل الأداة بشكل أصلي مع Claude Code، ومع أي أداة ذكاء اصطناعي أخرى على سطر الأوامر (Codex، Grok، ...) عبر أمر غلاف (wrapper) صغير.

## كيف تعمل الأداة

```
You type: "帮我重构这个函数,保持接口不变"
   │
   ├─ block mode  ──→ 🛡 Intercepted. Please rewrite it in English.
   │                   You: "please refactor this function"
   │                   LLM judge: ❌ score 55 — hint: missing "keep the interface unchanged"
   │                   You: "refactor this function but keep the interface unchanged"
   │                   LLM judge: ✅ score 93 — allowed through, pair logged
   │                   (type giveup at any point to get the English phrasing and continue)
   │
   ├─ warn mode  ──→ ⚠️ Warned, auto-translated to English, and logged
   ├─ log mode   ──→ Silently translated and logged
   └─ hits a stopword (e.g. "urgent"/"紧急") ──→ ⚡ block skipped, allowed straight through, still logged
```

يمرّ كل طلب معترَض بنفس خط المعالجة (pipeline):

1. **الكشف (Detect)** — هل النص غير إنجليزي؟ (`src/detect.js`، وانظر قسم [الإدخال بلغات متعددة](#multi-language-input) أدناه لمعرفة التعريف الدقيق لما يُحسب "غير إنجليزي").
2. **القرار (Decide)** — حسب الوضع الحالي (`block` / `warn` / `log`)، إما أن تُحجب الأداة الطلب وتطلب منك إعادة صياغته بالإنجليزية، أو تترجمه وتسمح بمروره مع ملاحظة توضيحية.
3. **الحكم (Judge)** — في وضع `block`، تتم مقارنة صياغتك الإنجليزية بالنص الأصلي بواسطة نموذج لغوي (LLM) للتحقق من التكافؤ الدلالي (ليس فقط "هل هذا إنجليزي؟"، بل "هل يحمل نفس المعنى؟"). إذا أسقطت صياغتك متطلبًا ما أو غيّرت النطاق، تحصل على تلميح وفرصة أخرى، وليس الترجمة الكاملة - فالهدف أن تتوصل إليها بنفسك.
4. **التسجيل (Log)** — يُضاف الزوج (الأصلي/الإنجليزي) إلى قائمة انتظار JSONL محلية لمراجعته لاحقًا.

الأداة مصممة على مبدأ "الفشل المفتوح" (fail-open): إذا فشل استدعاء النموذج اللغوي، أو انتهت مهلته، أو حدث خطأ داخل الـ hook نفسه، يُسمح بمرور المُدخل دون تعديل. هذه الأداة يجب ألا تكون أبدًا سبب توقف عملك الحقيقي.

## التثبيت

### الطريقة المُوصى بها: إضافة (plugin) لـ Claude Code

يُشحن المستودع كإضافة أصلية لـ Claude Code (`.claude-plugin/marketplace.json` + `.claude-plugin/plugin.json` + `hooks/hooks.json`)، فلا حاجة لاستنساخ المستودع أو تنفيذ `npm link`:

```
/plugin marketplace add s546126/english-by-default
/plugin install english-by-default@english-by-default
```

هذا كل ما في الأمر - يتم ربط hook الخاص بـ `UserPromptSubmit` تلقائيًا، ويبدأ بفحص طلباتك فورًا في وضع `block` (الوضع الافتراضي).

### التثبيت اليدوي / كأداة سطر أوامر مستقلة

ما زلت بحاجة لهذا المسار إذا أردت استخدام أمر `ebd` مباشرة - فأوامر مثل `ebd x`، و`ebd gate`، و`ebd quiz`، و`ebd review`، و`ebd feynman`، و`ebd web`، وغيرها تعمل خارج Claude Code تمامًا، وتثبيت الإضافة أعلاه لا يضع ملف تنفيذي `ebd` على متغير `PATH` لديك.

```bash
git clone https://github.com/s546126/english-by-default.git
cd english-by-default
npm link                  # gives you the global `ebd` command
ebd install claude-code    # writes a UserPromptSubmit hook into ~/.claude/settings.json
```

الأمر `ebd uninstall claude-code` يلغي تثبيت الـ hook.

### النموذج اللغوي المستخدم في الخلفية (LLM backend)

تعتمد الترجمة والحكم الدلالي افتراضيًا على أداة `claude -p --model haiku` المحلية. يمكنك توجيه `llm.command` في ملف `~/.english-by-default/config.json` إلى أي أداة سطر أوامر تكون آخر معطياتها (argument) هي النص المطلوب، ويكون ناتج stdout هو الإجابة (مثل `codex exec`).

## الاستخدام

```bash
ebd mode block|warn|log     # strength: block | warn | log-and-pass (default: block)
ebd stopwords list|add <w>|rm <w>   # emergency words: matching one skips the block entirely
ebd list [n]                 # show the last n original/English pairs (default 10)
ebd stats                    # queue + review stats

ebd quiz [n]                 # pop quiz: see the original, write the English from memory, LLM scores it (default 5)
ebd review                   # spaced-repetition review: only due entries, pass promotes the interval, fail demotes it
ebd feynman                  # Feynman technique: explain the idea in the simplest English you can, LLM probes for gaps
```

### أدوات أخرى على سطر الأوامر (Codex / Grok / أي أداة)

يحصل Claude Code على hook أصلي مخصص. أما كل الأدوات الأخرى فتعمل عبر غلاف (wrapper) يفحص المعطيات غير الإنجليزية قبل استدعاء الأداة الحقيقية:

```bash
ebd x codex exec "帮我把这个接口改成分页的"
ebd x grok "解释一下这段代码"
ebd gate "帮我写个爬虫"     # non-interactive: prints the English version and logs the pair, good for scripts
```

يفحص `ebd x` فقط النص الممرَّر إليه *كمعطيات على سطر الأوامر* - فهو لا يستطيع رؤية أي شيء تكتبه داخل واجهة تفاعلية (REPL/TUI) بعد بدء تشغيل الأداة المُغلَّفة (ذلك الإدخال يذهب مباشرة إلى stdin الخاصة بالعملية الفرعية). راجع [`docs/ADAPTERS.md`](../ADAPTERS.md) لمعرفة التفاصيل الخاصة بكل أداة، بما في ذلك أي أدوات سطر أوامر تملك آلية hook أصلية شبيهة بـ `UserPromptSubmit` يمكنك توجيهها مباشرة إلى `hooks/claude-code-hook.js` بدلًا من استخدام الغلاف.

## الإدخال بلغات متعددة

كشف اللغة هنا مبني عمدًا على **نطاقات يونيكود للأبجديات (Unicode script)**، وليس على مصنّف لغوي عام. يضع `src/detect.js` علامة على أي نص باعتباره غير إنجليزي عندما يحتوي على حرفين أو أكثر من نطاق يونيكود غير لاتيني (CJK، الكانا، الهانغل، السيريلية، العبرية، العربية، الديفاناغري، التايلاندية). يُزال النص الموجود داخل كتل الأكواد المسيَّجة (fenced code blocks) أو مقاطع الكود المضمّنة (inline code) قبل عملية العد، لذا فإن لصق مقتطف كود يحتوي على تعليقات أو نصوص غير إنجليزية لا يُفعّل البوابة.

بالنسبة للنص الذي يُفعّل البوابة فعلًا، تقوم دالة `detectLanguage()` في `src/detect.js` بتضييق تحديد *أي لغة* هي، اعتمادًا فقط على نطاق يونيكود المستخدم، ويستعمل `src/i18n.js` ذلك لتوطين (localize) رسائل الحجب/التحذير/التلميح الخاصة بالأداة نفسها (يبقى الطلب مُترجَمًا إلى الإنجليزية بواسطة النموذج اللغوي بغض النظر عن ذلك - هذا يؤثر فقط على اللغة التي تُعرَض بها رسائل الأداة). اللغات المدعومة:

| الرمز | اللغة | النطاق المستخدَم للكشف عنها |
|---|---|---|
| `zh` | الصينية | الأيديوغرامات الصينية (CJK)، بدون وجود حروف الكانا |
| `ja` | اليابانية | وجود حروف الهيراغانا/الكاتاكانا (يميّزها عن `zh`، لأن الكانجي يتداخل مع الصينية) |
| `ko` | الكورية | أبجدية الهانغل |
| `ru` | الروسية | الأبجدية السيريلية |
| `he` | العبرية | الأبجدية العبرية |
| `ar` | العربية | الأبجدية العربية |
| `hi` | الهندية | أبجدية الديفاناغري |
| `th` | التايلاندية | الأبجدية التايلاندية |

**اللغات ذات الأبجدية اللاتينية غير مكتشَفة أو محجوبة عمدًا.** الإسبانية والفرنسية والألمانية والبرتغالية والإيطالية والهولندية وما شابهها تستخدم نفس الحروف اللاتينية التي تستخدمها الإنجليزية، لذا فإن فحصًا قائمًا على نطاق الأبجدية لا يستطيع تمييزها عن الإنجليزية (أو عن كلمة إنجليزية مُستعارة بحرف منقّط مثل "café"). تمييز هذه اللغات يتطلب قائمة كلمات أو معيارًا إحصائيًا، وهو أمر خارج نطاق هذه الوظيفة بتصميمها - إنه قيد معروف ومقصود، وليس خللًا. الطلبات بهذه اللغات تمر حاليًا دون فحص.

## لوحة التحكم على الويب (Web dashboard)

```bash
ebd web [port]   # default port 4173
```

يشغّل هذا الأمر لوحة تحكم محلية للقراءة فقط (باستخدام وحدة `http` القياسية في Node، بدون أي اعتماديات) تعرض:

- إجمالي عدد الأزواج المسجَّلة، ومتوسط درجة التذكّر/المراجعة، وعدد المُدخلات المستحقة للمراجعة، وتوزيعًا حسب الوضع (`rewrite` / `warn` / `log` / `giveup` / `stopword` / ...).
- الكلمات/العبارات الأكثر تكرارًا من النصوص الأصلية المسجَّلة لديك (بالاعتماد على الكلمات للنصوص اللاتينية، وعلى ثنائيات الأحرف المتجاورة (character-bigram) للنصوص الشبيهة بـ CJK، لأنها لا تملك حدودًا طبيعية بين الكلمات دون قاموس).
- التعبيرات الأخيرة التي رصدها حكم الطبيعية اللغوية على أنها **غير** أصيلة الصياغة (`natural: false`)، مع الصياغة الأكثر أصالة التي يقترحها النموذج اللغوي.

**ملاحظة أمنية:** الخادم مرتبط بـ `127.0.0.1` فقط ولا يملك أي آلية مصادقة (authentication). وهو مخصص للعرض من نفس الجهاز فقط. لا تضعه خلف منفذ عام، أو نفق (tunnel)، أو أي نوع من الوكيل العكسي (reverse proxy) يعرّضه للشبكة - فأي شخص يستطيع الوصول إليه يمكنه قراءة كامل سجل طلباتك.

## تفاصيل إضافية

- **الحكم الدلالي (Semantic judging)** — في وضع `block`، تتم مقارنة صياغتك الإنجليزية بالنص الأصلي من قِبل النموذج اللغوي من ناحية المعنى، وليس فقط اللغة (الحد الافتراضي: درجة ≥ 70، ويمكن ضبطه عبر `judgeThreshold` في `config.json`). أي عدم تطابق يمنحك تلميحًا، لا الترجمة الكاملة أبدًا، لذا عليك التوصل إليها بنفسك.
- **giveup** — اكتب `giveup` عند أي طلب حجب/إعادة صياغة للتخلي عن المحاولة؛ تُعرَض لك الترجمة الإنجليزية ويُمرَّر النص، ويظل الزوج مسجَّلًا للمراجعة.
- **قائمة الانتظار (Queue)** — تُخزَّن في `~/.english-by-default/queue.jsonl`، وكل سطر يمثّل كائن JSON واحدًا، بحد أقصى افتراضي هو آخر 1000 مُدخَل (`queueSize` في `config.json`).
- **الكلمات الاستثنائية (Stopwords)** — المُدخلات التي تمر عبر تطابق كلمة استثنائية تُسجَّل دون ترجمة متزامنة (حتى لا يتباطأ أي أمر طارئ حقيقي)؛ وتُملأ النسخة الإنجليزية لاحقًا وبشكل كسول (lazily) في المرة القادمة التي تراجع أو تختبر فيها ذلك المُدخل.
- **الفشل المفتوح (Fail-open)** — أي فشل في استدعاء النموذج اللغوي، أو انتهاء مهلة، أو استثناء داخل الـ hook، يسمح بمرور الطلب دون تعديل. هذه الأداة يجب ألا تكون أبدًا سبب حجب عمل مشروع.
- **بدون أي اعتماديات (Zero dependencies)**، وتتطلب Node.js بإصدار 18 أو أحدث.
