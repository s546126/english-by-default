# English by Default

AI CLI के लिए एक "इंग्लिश गेट"। यह गैर-अंग्रेज़ी प्रॉम्प्ट को मॉडल तक पहुँचने से पहले ही रोक लेता है, आपसे उसे ख़ुद अंग्रेज़ी में दोबारा लिखवाता है (या आप हार मान लें तो अनुवाद दिखाकर आगे बढ़ने देता है), और इस तरह इकट्ठा हुए हर मूल-भाषा/अंग्रेज़ी जोड़े को स्पेस्ड-रिपिटिशन और फाइनमैन तकनीक के अभ्यास में बदल देता है।

Claude Code के साथ यह मूल रूप से (natively) काम करता है, और किसी भी दूसरे AI CLI (Codex, Grok, ...) के साथ एक छोटे-से wrapper कमांड के ज़रिए।

## यह कैसे काम करता है

```
You type: "帮我重构这个函数,保持接口不变"
   │
   ├─ block mode  ──→ 🛡 Intercepted. Please rewrite it in English.
   │                   You: "please refactor this function"
   │                   LLM judge: ❌ score 55 — hint: missing "keep the interface unchanged"
   │                   You: "refactor this function but keep the interface unchanged"
   │                   LLM judge: ✅ score 93 — allowed through, pair logged
   │                   (type giveup at any point to get the English phrasing and continue)
   │
   ├─ warn mode  ──→ ⚠️ Warned, auto-translated to English, and logged
   ├─ log mode   ──→ Silently translated and logged
   └─ hits a stopword (e.g. "urgent"/"紧急") ──→ ⚡ block skipped, allowed straight through, still logged
```

रोका गया हर प्रॉम्प्ट एक ही पाइपलाइन से होकर गुज़रता है:

1. **पहचान (Detect)** — क्या यह टेक्स्ट ग़ैर-अंग्रेज़ी है? (`src/detect.js`; ठीक-ठीक क्या चीज़ "ग़ैर-अंग्रेज़ी" गिनी जाती है, इसके लिए नीचे [बहुभाषी इनपुट](#बहुभाषी-इनपुट) सेक्शन देखें।)
2. **फ़ैसला (Decide)** — मौजूदा मोड (`block` / `warn` / `log`) के आधार पर, या तो प्रॉम्प्ट को रोककर अंग्रेज़ी में दोबारा लिखने को कहा जाता है, या उसे अनुवाद करके एक नोट के साथ आगे जाने दिया जाता है।
3. **मूल्यांकन (Judge)** — `block` मोड में आपकी लिखी अंग्रेज़ी रीराइट की तुलना LLM मूल टेक्स्ट से सिमैंटिक स्तर पर करता है (सिर्फ़ यह नहीं कि "क्या यह अंग्रेज़ी है", बल्कि यह कि "क्या इसका मतलब वही है")। अगर रीराइट में कोई शर्त छूट जाए या दायरा बदल जाए, तो आपको पूरा अनुवाद नहीं बल्कि एक संकेत और एक और मौक़ा मिलता है — मक़सद यही है कि आप इसे ख़ुद लिखकर निकालें।
4. **रिकॉर्ड (Log)** — मूल और अंग्रेज़ी वाला जोड़ा बाद की समीक्षा के लिए एक लोकल JSONL क्यू में जोड़ दिया जाता है।

यह डिज़ाइन से ही fail-open है: अगर LLM कॉल फेल हो जाए, टाइमआउट हो जाए, या hook में ख़ुद कोई एक्सेप्शन आ जाए, तो इनपुट को बिना किसी बदलाव के आगे जाने दिया जाता है। यह टूल कभी भी आपके असली काम को रोकने की वजह नहीं बनना चाहिए।

## इंस्टॉल

### अनुशंसित तरीक़ा: Claude Code प्लगइन

यह रिपॉज़िटरी एक नेटिव Claude Code प्लगइन के रूप में शिप होती है (`.claude-plugin/marketplace.json` + `.claude-plugin/plugin.json` + `hooks/hooks.json`), इसलिए इसे क्लोन करने या `npm link` चलाने की कोई ज़रूरत नहीं:

```
/plugin marketplace add s546126/english-by-default
/plugin install english-by-default@english-by-default
```

बस इतना ही — `UserPromptSubmit` hook अपने-आप जुड़ जाता है और तुरंत `block` मोड (जो डिफ़ॉल्ट है) में आपके प्रॉम्प्ट्स को गेट करना शुरू कर देता है।

### मैनुअल / स्टैंडअलोन CLI

अगर आप `ebd` कमांड को सीधे इस्तेमाल करना चाहते हैं — `ebd x`, `ebd gate`, `ebd quiz`, `ebd review`, `ebd feynman`, `ebd web` वगैरह, जो Claude Code से पूरी तरह अलग होकर भी काम करते हैं — तो यह वाला रास्ता अब भी आपके काम आएगा, क्योंकि ऊपर वाला प्लगइन इंस्टॉल आपके `PATH` में `ebd` बाइनरी नहीं डालता।

```bash
git clone https://github.com/s546126/english-by-default.git
cd english-by-default
npm link                  # gives you the global `ebd` command
ebd install claude-code    # writes a UserPromptSubmit hook into ~/.claude/settings.json
```

`ebd uninstall claude-code` इस hook इंस्टॉलेशन को वापस पलट (undo) कर देता है।

### LLM बैकएंड

अनुवाद और मूल्यांकन दोनों डिफ़ॉल्ट रूप से लोकल `claude -p --model haiku` CLI इस्तेमाल करते हैं। आप `~/.english-by-default/config.json` में `llm.command` को किसी भी ऐसे CLI की तरफ़ पॉइंट कर सकते हैं जहाँ आख़िरी आर्ग्युमेंट प्रॉम्प्ट हो और stdout जवाब हो (जैसे `codex exec`)।

## उपयोग

```bash
ebd mode block|warn|log     # strength: block | warn | log-and-pass (default: block)
ebd stopwords list|add <w>|rm <w>   # emergency words: matching one skips the block entirely
ebd list [n]                 # show the last n original/English pairs (default 10)
ebd stats                    # queue + review stats

ebd quiz [n]                 # pop quiz: see the original, write the English from memory, LLM scores it (default 5)
ebd review                   # spaced-repetition review: only due entries, pass promotes the interval, fail demotes it
ebd feynman                  # Feynman technique: explain the idea in the simplest English you can, LLM probes for gaps
```

### दूसरे CLI (Codex / Grok / कोई भी)

Claude Code को एक नेटिव hook मिलता है। बाक़ी सब एक wrapper के ज़रिए चलते हैं, जो असली टूल को कॉल करने से पहले ग़ैर-अंग्रेज़ी आर्ग्युमेंट्स को गेट करता है:

```bash
ebd x codex exec "帮我把这个接口改成分页的"
ebd x grok "解释一下这段代码"
ebd gate "帮我写个爬虫"     # non-interactive: prints the English version and logs the pair, good for scripts
```

`ebd x` सिर्फ़ वही टेक्स्ट देखता है जो उसे *कमांड-लाइन आर्ग्युमेंट के रूप में* दिया जाता है — wrap किए गए टूल के शुरू होने के बाद आप किसी इंटरैक्टिव REPL/TUI में जो भी टाइप करते हैं, वह इसे बिल्कुल नहीं दिखता (वह इनपुट सीधे चाइल्ड प्रोसेस के stdin में चला जाता है)। हर टूल की विस्तृत जानकारी के लिए [`docs/ADAPTERS.md`](../ADAPTERS.md) देखें — इसमें यह भी बताया गया है कि किन CLI के पास अपना ख़ुद का नेटिव, `UserPromptSubmit`-जैसा hook है, जिसे आप wrapper इस्तेमाल करने की बजाय सीधे `hooks/claude-code-hook.js` पर पॉइंट कर सकते हैं।

## बहुभाषी इनपुट

भाषा की पहचान जान-बूझकर **Unicode स्क्रिप्ट पर आधारित** रखी गई है, न कि कोई आम भाषा-वर्गीकरणकर्ता (classifier)। `src/detect.js` किसी टेक्स्ट को तभी ग़ैर-अंग्रेज़ी मानता है जब उसमें ग़ैर-लैटिन Unicode रेंज (CJK, काना, हांगुल, सिरिलिक, हिब्रू, अरबी, देवनागरी, थाई) के दो या ज़्यादा अक्षर हों। गिनती से पहले फेंस्ड कोड ब्लॉक और इनलाइन कोड स्पैन के अंदर का टेक्स्ट हटा दिया जाता है, इसलिए ग़ैर-अंग्रेज़ी कमेंट या स्ट्रिंग वाला कोई स्निपेट पेस्ट करने भर से गेट ट्रिगर नहीं होता।

जिस टेक्स्ट पर गेट ट्रिगर हो जाता है, उसके लिए `src/detect.js` का `detectLanguage()` सिर्फ़ शामिल Unicode स्क्रिप्ट के आधार पर आगे यह भी तय करता है कि यह *कौन-सी भाषा* है, और `src/i18n.js` इसी जानकारी से block/warn/hint मैसेज ख़ुद को स्थानीय भाषा में दिखाता है (प्रॉम्प्ट फिर भी हर हाल में LLM द्वारा अंग्रेज़ी में अनूदित होता है — इसका असर सिर्फ़ इतना पड़ता है कि टूल के अपने मैसेज किस भाषा में दिखते हैं)। समर्थित भाषाएँ:

| कोड | भाषा | पहचान के लिए इस्तेमाल की गई लिपि |
|---|---|---|
| `zh` | चीनी | CJK आइडियोग्राफ़, बिना काना के |
| `ja` | जापानी | हिरागाना/काताकाना की मौजूदगी (यह `zh` से अलग करने में मदद करता है, क्योंकि कांजी चीनी लिपि से ओवरलैप करती है) |
| `ko` | कोरियाई | हांगुल |
| `ru` | रूसी | सिरिलिक |
| `he` | हिब्रू | हिब्रू लिपि |
| `ar` | अरबी | अरबी लिपि |
| `hi` | हिन्दी | देवनागरी |
| `th` | थाई | थाई लिपि |

**लैटिन-लिपि वाली भाषाओं को जान-बूझकर न तो पहचाना जाता है, न रोका जाता है।** स्पैनिश, फ़्रेंच, जर्मन, पुर्तगाली, इतालवी, डच जैसी भाषाएँ अंग्रेज़ी जैसे ही लैटिन अक्षर इस्तेमाल करती हैं, इसलिए सिर्फ़ लिपि-रेंज की जाँच इन्हें अंग्रेज़ी से (या "café" जैसे किसी accent वाले अंग्रेज़ी लोनवर्ड से) अलग नहीं बता सकती। इन्हें अलग पहचानने के लिए एक वर्डलिस्ट या कोई स्टैटिस्टिकल हेयुरिस्टिक चाहिए होगी, जो डिज़ाइन के हिसाब से इस फ़ंक्शन के दायरे से बिल्कुल बाहर है — यह एक जानी-पहचानी और जान-बूझकर रखी गई सीमा है, कोई बग नहीं। फ़िलहाल इन भाषाओं के प्रॉम्प्ट बिना किसी रोक-टोक के सीधे आगे निकल जाते हैं।

## वेब डैशबोर्ड

```bash
ebd web [port]   # default port 4173
```

एक लोकल, रीड-ओनली डैशबोर्ड शुरू करता है (सादा Node `http`, कोई डिपेंडेंसी नहीं), जो यह दिखाता है:

- रिकॉर्ड किए गए जोड़ों की कुल संख्या, औसत रीकॉल/रिव्यू स्कोर, कितनी एंट्रीज़ रिव्यू के लिए ड्यू हैं, और मोड के हिसाब से (`rewrite` / `warn` / `log` / `giveup` / `stopword` / ...) एक ब्रेकडाउन।
- आपकी लॉग की गई मूल एंट्रीज़ से निकाले गए बार-बार आने वाले शब्द/वाक्यांश (लैटिन-लिपि टेक्स्ट के लिए वर्ड-आधारित, CJK-जैसे टेक्स्ट के लिए कैरेक्टर-बाइग्राम-आधारित, क्योंकि बिना डिक्शनरी के उसमें शब्दों की कोई प्राकृतिक सीमा नहीं होती)।
- हाल की वे अभिव्यक्तियाँ जिन्हें naturalness judge ने **ग़ैर**-मुहावरेदार (`natural: false`) चिह्नित किया है, साथ में LLM का सुझाया हुआ ज़्यादा नेटिव फ़्रेज़िंग।

**सुरक्षा नोट:** यह सर्वर सिर्फ़ `127.0.0.1` पर बाइंड होता है और इसमें कोई ऑथेंटिकेशन नहीं है। यह उसी मशीन से देखे जाने के लिए बना है। इसे किसी पब्लिक पोर्ट, टनल, या ऐसे किसी भी रिवर्स प्रॉक्सी के पीछे मत रखिए जो इसे नेटवर्क पर एक्सपोज़ कर दे — जो भी इस तक पहुँच सकता है, वह आपकी पूरी प्रॉम्प्ट हिस्ट्री पढ़ सकता है।

## विवरण

- **सिमैंटिक मूल्यांकन (Semantic judging)** — `block` मोड में आपकी अंग्रेज़ी रीराइट की LLM द्वारा मूल टेक्स्ट से अर्थ के आधार पर तुलना होती है, सिर्फ़ भाषा के आधार पर नहीं (डिफ़ॉल्ट थ्रेशोल्ड: स्कोर ≥ 70, `config.json` में `judgeThreshold` से कॉन्फ़िगर करने योग्य)। मेल न खाने पर सिर्फ़ एक संकेत मिलता है, कभी पूरा अनुवाद नहीं, ताकि आपको ख़ुद सोचकर निकालना पड़े।
- **giveup** — किसी भी block/rewrite प्रॉम्प्ट पर `giveup` टाइप करके बाहर निकल जाइए; आपको अंग्रेज़ी अनुवाद दिखा दिया जाता है और उसे आगे भेज दिया जाता है, और यह जोड़ा फिर भी समीक्षा के लिए लॉग हो जाता है।
- **क्यू (Queue)** — `~/.english-by-default/queue.jsonl` में सेव होती है, हर लाइन में एक JSON ऑब्जेक्ट, डिफ़ॉल्ट रूप से सिर्फ़ हाल की 1000 एंट्रीज़ तक सीमित (`config.json` में `queueSize`)।
- **स्टॉपवर्ड (आपातकालीन शब्द)** — स्टॉपवर्ड मैच से आगे जाने दी गई एंट्रीज़ बिना तुरंत अनुवाद के लॉग होती हैं (ताकि किसी असली इमरजेंसी में कुछ भी धीमा न पड़े); उस एंट्री की अंग्रेज़ी अगली बार जब आप उसकी समीक्षा या क्विज़ करते हैं, तब लेज़ी तरीक़े से भर दी जाती है।
- **फ़ेल-ओपन (Fail-open)** — कोई भी LLM कॉल फेल्योर, टाइमआउट, या hook एक्सेप्शन प्रॉम्प्ट को बिना बदले आगे जाने देता है। यह टूल कभी भी जायज़ काम को रोकने की वजह नहीं बनना चाहिए।
- **ज़ीरो डिपेंडेंसी**, Node.js >= 18।
