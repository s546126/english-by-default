# English by Default

"ด่านภาษาอังกฤษ" สำหรับ AI CLI ตัวไหนก็ได้ มันจะสกัดพรอมป์ที่ไม่ใช่ภาษาอังกฤษไว้ก่อนที่จะไปถึงโมเดล บังคับให้คุณเขียนมันใหม่เป็นภาษาอังกฤษด้วยตัวเอง (หรือจะยอมแพ้แล้วดูคำแปลก็ได้) แล้วนำทุกคู่ประโยคที่เก็บรวบรวมมาได้ แปลงเป็นแบบฝึกหัดทบทวนแบบเว้นระยะ (spaced repetition) และเทคนิคไฟน์แมน (Feynman technique)

ใช้งานร่วมกับ Claude Code ได้แบบเนทีฟ และใช้กับ AI CLI ตัวอื่นๆ (Codex, Grok, ...) ได้เช่นกันผ่านคำสั่ง wrapper เล็กๆ ตัวหนึ่ง

## วิธีการทำงาน

```
You type: "帮我重构这个函数,保持接口不变"
   │
   ├─ block mode  ──→ 🛡 Intercepted. Please rewrite it in English.
   │                   You: "please refactor this function"
   │                   LLM judge: ❌ score 55 — hint: missing "keep the interface unchanged"
   │                   You: "refactor this function but keep the interface unchanged"
   │                   LLM judge: ✅ score 93 — allowed through, pair logged
   │                   (type giveup at any point to get the English phrasing and continue)
   │
   ├─ warn mode  ──→ ⚠️ Warned, auto-translated to English, and logged
   ├─ log mode   ──→ Silently translated and logged
   └─ hits a stopword (e.g. "urgent"/"紧急") ──→ ⚡ block skipped, allowed straight through, still logged
```

พรอมป์ทุกอันที่ถูกสกัดไว้จะผ่านขั้นตอนเดียวกันนี้:

1. **ตรวจจับ (Detect)** — เช็กว่าข้อความนี้ไม่ใช่ภาษาอังกฤษหรือเปล่า (`src/detect.js` ดูรายละเอียดว่าอะไรถึงนับว่า "ไม่ใช่ภาษาอังกฤษ" ได้ในหัวข้อ [การป้อนข้อมูลหลายภาษา](#การป้อนข้อมูลหลายภาษา) ด้านล่าง)
2. **ตัดสินใจ (Decide)** — อิงตามโหมดปัจจุบัน (`block` / `warn` / `log`) แล้วเลือกว่าจะสกัดไว้แล้วขอให้เขียนใหม่เป็นภาษาอังกฤษ หรือแปลให้แล้วปล่อยผ่านพร้อมหมายเหตุ
3. **ตัดสิน (Judge)** — ในโหมด `block` LLM จะเทียบข้อความภาษาอังกฤษที่คุณเขียนใหม่กับต้นฉบับ เพื่อดูว่าความหมายตรงกันหรือไม่ (ไม่ใช่แค่เช็กว่า "นี่เป็นภาษาอังกฤษไหม" แต่เช็กว่า "ความหมายเหมือนกันไหม") ถ้าเขียนใหม่แล้วตกหล่นเงื่อนไขบางอย่าง หรือเปลี่ยนขอบเขตไป คุณจะได้คำใบ้พร้อมโอกาสลองใหม่อีกครั้ง ไม่ใช่คำแปลเต็มๆ — เป้าหมายคือบังคับให้คุณคิดออกมาเอง
4. **บันทึก (Log)** — คู่ต้นฉบับ/ภาษาอังกฤษจะถูกต่อท้ายเข้าไปในคิว JSONL บนเครื่อง เพื่อเก็บไว้ทบทวนภายหลัง

ออกแบบมาให้เป็นแบบ fail-open (พังแล้วปล่อยผ่าน) โดยตั้งใจ: ถ้าการเรียก LLM ล้มเหลว หมดเวลา หรือตัว hook เองเกิด error ข้อความจะถูกปล่อยผ่านไปโดยไม่แก้ไขอะไรเลย เครื่องมือนี้ไม่ควรเป็นสาเหตุที่ทำให้งานจริงของคุณติดขัด

## การติดตั้ง

### แนะนำ: ปลั๊กอิน Claude Code

รีโพนี้มาพร้อมโครงสร้างปลั๊กอิน Claude Code แบบเนทีฟอยู่แล้ว (`.claude-plugin/marketplace.json` + `.claude-plugin/plugin.json` + `hooks/hooks.json`) จึงไม่ต้อง clone หรือรัน `npm link` เลย:

```
/plugin marketplace add s546126/english-by-default
/plugin install english-by-default@english-by-default
```

แค่นี้ก็เสร็จ — hook `UserPromptSubmit` จะถูกต่อสายให้อัตโนมัติ และเริ่มคัดกรองพรอมป์ของคุณด้วยโหมด `block` (ค่าเริ่มต้น) ทันที

### ติดตั้งเอง / ใช้เป็น CLI แยกเดี่ยว

ถ้าคุณอยากใช้คำสั่ง `ebd` โดยตรง — `ebd x`, `ebd gate`, `ebd quiz`, `ebd review`, `ebd feynman`, `ebd web` ฯลฯ ซึ่งทำงานได้เองนอก Claude Code ทั้งหมด — คุณยังต้องติดตั้งด้วยวิธีนี้ เพราะการติดตั้งแบบปลั๊กอินด้านบนจะไม่ได้เอาไบนารี `ebd` มาใส่ไว้ใน `PATH` ของคุณ

```bash
git clone https://github.com/s546126/english-by-default.git
cd english-by-default
npm link                  # gives you the global `ebd` command
ebd install claude-code    # writes a UserPromptSubmit hook into ~/.claude/settings.json
```

`ebd uninstall claude-code` จะย้อนกลับการติดตั้ง hook นี้

### LLM แบ็กเอนด์

การแปลและการตัดสินใช้ CLI ในเครื่อง `claude -p --model haiku` เป็นค่าเริ่มต้น คุณสามารถชี้ `llm.command` ใน `~/.english-by-default/config.json` ไปยัง CLI ตัวไหนก็ได้ ขอแค่ argument สุดท้ายเป็นพรอมป์ และ stdout เป็นคำตอบ (เช่น `codex exec`)

## วิธีใช้งาน

```bash
ebd mode block|warn|log     # strength: block | warn | log-and-pass (default: block)
ebd stopwords list|add <w>|rm <w>   # emergency words: matching one skips the block entirely
ebd list [n]                 # show the last n original/English pairs (default 10)
ebd stats                    # queue + review stats

ebd quiz [n]                 # pop quiz: see the original, write the English from memory, LLM scores it (default 5)
ebd review                   # spaced-repetition review: only due entries, pass promotes the interval, fail demotes it
ebd feynman                  # Feynman technique: explain the idea in the simplest English you can, LLM probes for gaps
```

### CLI อื่นๆ (Codex / Grok / อะไรก็ได้)

Claude Code มี hook แบบเนทีฟให้ใช้อยู่แล้ว ส่วนเครื่องมืออื่นๆ ทั้งหมดจะวิ่งผ่าน wrapper ที่คอยคัดกรอง argument ที่ไม่ใช่ภาษาอังกฤษ ก่อนจะเรียกใช้เครื่องมือจริง:

```bash
ebd x codex exec "帮我把这个接口改成分页的"
ebd x grok "解释一下这段代码"
ebd gate "帮我写个爬虫"     # non-interactive: prints the English version and logs the pair, good for scripts
```

`ebd x` จะตรวจสอบเฉพาะข้อความที่ส่งเข้ามาในรูป *command-line argument* เท่านั้น — มันมองไม่เห็นอะไรก็ตามที่คุณพิมพ์เข้าไปใน REPL/TUI แบบโต้ตอบ หลังจากเครื่องมือที่ถูกห่ออยู่เริ่มทำงานแล้ว (input ส่วนนั้นวิ่งตรงเข้า stdin ของ child process เลย) ดูรายละเอียดของแต่ละเครื่องมือได้ที่ [`docs/ADAPTERS.md`](../ADAPTERS.md) รวมถึงว่า CLI ตัวไหนมี hook แบบ `UserPromptSubmit` เนทีฟของตัวเอง ที่คุณชี้ไปที่ `hooks/claude-code-hook.js` ได้โดยตรง แทนที่จะต้องใช้ wrapper

## การป้อนข้อมูลหลายภาษา

การตรวจจับภาษาถูกออกแบบมาให้ทำงานแบบ**อิงช่วง Unicode ของตัวอักษร (script)** โดยตั้งใจ ไม่ใช่ตัวจำแนกภาษาแบบทั่วไป `src/detect.js` จะตีตราข้อความว่าไม่ใช่ภาษาอังกฤษ เมื่อพบตัวอักษรตั้งแต่ 2 ตัวขึ้นไปจากช่วง Unicode ที่ไม่ใช่ละติน (CJK, คานะ, ฮันกึล, ซีริลลิก, ฮีบรู, อาหรับ, เทวนาครี, ไทย) ข้อความที่อยู่ใน fenced code block หรือ inline code จะถูกตัดออกก่อนนับ ดังนั้นการวางโค้ดที่มีคอมเมนต์หรือสตริงภาษาอื่นแปะเข้าไปด้วย จะไม่ทำให้ระบบสกัดทำงาน

สำหรับข้อความที่ทำให้ระบบสกัดทำงานจริง ฟังก์ชัน `detectLanguage()` ใน `src/detect.js` จะเจาะลึกต่อไปอีกว่ามัน*เป็นภาษาไหน* โดยดูจากช่วง Unicode ของตัวอักษรที่เจอล้วนๆ แล้ว `src/i18n.js` จะใช้ข้อมูลนี้ไปแปลข้อความ block/warn/hint ของตัวเครื่องมือเองให้เป็นภาษานั้น (ตัวพรอมป์เองยังคงถูก LLM แปลเป็นภาษาอังกฤษเหมือนเดิมไม่ว่ากรณีไหน — ตรงนี้มีผลแค่กับภาษาที่ใช้แสดงข้อความของตัวเครื่องมือเองเท่านั้น) ภาษาที่รองรับ:

| รหัส | ภาษา | ตัวอักษรที่ใช้ตรวจจับ |
|---|---|---|
| `zh` | จีน | ตัวอักษรจีน (CJK ideographs) โดยไม่มีคานะปน |
| `ja` | ญี่ปุ่น | มีฮิรางานะ/คาตากานะปรากฏ (ใช้แยกจาก `zh` เพราะคันจิทับซ้อนกับตัวอักษรจีน) |
| `ko` | เกาหลี | ฮันกึล (Hangul) |
| `ru` | รัสเซีย | ซีริลลิก |
| `he` | ฮีบรู | ตัวอักษรฮีบรู |
| `ar` | อาหรับ | ตัวอักษรอาหรับ |
| `hi` | ฮินดี | เทวนาครี |
| `th` | ไทย | ตัวอักษรไทย |

**ภาษาที่ใช้ตัวอักษรละตินจะไม่ถูกตรวจจับหรือสกัด โดยตั้งใจ** สเปน ฝรั่งเศส เยอรมัน โปรตุเกส อิตาลี ดัตช์ และภาษาอื่นๆ ที่คล้ายกัน ใช้ตัวอักษรละตินตัวเดียวกับภาษาอังกฤษ การเช็กแค่ช่วงตัวอักษรจึงแยกมันออกจากภาษาอังกฤษไม่ได้ (หรือแยกจากคำยืมภาษาอังกฤษที่มีเครื่องหมายเน้นเสียงอย่าง "café" ก็ไม่ได้เหมือนกัน) การจะแยกภาษาพวกนี้ออกจากกันได้ต้องใช้ wordlist หรือฮิวริสติกเชิงสถิติ ซึ่งอยู่นอกขอบเขตของฟังก์ชันนี้โดยตั้งใจตั้งแต่แรก — นี่คือข้อจำกัดที่รู้อยู่แล้วและตั้งใจให้เป็นแบบนั้น ไม่ใช่บั๊ก พรอมป์ในภาษาเหล่านี้ตอนนี้จึงยังผ่านไปได้โดยไม่ถูกสกัด

## แดชบอร์ดบนเว็บ

```bash
ebd web [port]   # default port 4173
```

เปิดแดชบอร์ดแบบอ่านอย่างเดียวในเครื่อง (ใช้ `http` เปล่าๆ ของ Node ไม่มี dependency ใดๆ) ซึ่งแสดง:

- จำนวนคู่ประโยคที่บันทึกไว้ทั้งหมด คะแนนความจำ/ทบทวนเฉลี่ย จำนวนรายการที่ถึงกำหนดทบทวน และสรุปแยกตามโหมด (`rewrite` / `warn` / `log` / `giveup` / `stopword` / ...)
- คำ/วลีที่ใช้บ่อย ดึงมาจากต้นฉบับที่คุณบันทึกไว้ (นับตามคำสำหรับข้อความตัวอักษรละติน และนับตาม character-bigram สำหรับข้อความแนว CJK เพราะไม่มีขอบเขตคำตามธรรมชาติถ้าไม่มีพจนานุกรม)
- สำนวนล่าสุดที่ตัวตัดสินความเป็นธรรมชาติตีตราว่า **ไม่** ค่อยเป็นสำนวน (`natural: false`) พร้อมสำนวนที่ LLM แนะนำว่าเจ้าของภาษาน่าจะพูดแบบไหนมากกว่า

**ข้อควรระวังด้านความปลอดภัย:** เซิร์ฟเวอร์นี้ bind กับ `127.0.0.1` เท่านั้น และไม่มีระบบยืนยันตัวตนใดๆ ตั้งใจให้ดูได้จากเครื่องเดียวกันเท่านั้น อย่าเอาไปเปิดพอร์ตสาธารณะ เปิด tunnel หรือวางไว้หลัง reverse proxy แบบไหนก็ตามที่จะเปิดให้เข้าถึงจากเครือข่ายภายนอกได้ — ใครก็ตามที่เข้าถึงมันได้จะอ่านประวัติพรอมป์ทั้งหมดของคุณได้เลย

## รายละเอียดเพิ่มเติม

- **การตัดสินเชิงความหมาย** — ในโหมด `block` ข้อความภาษาอังกฤษที่คุณเขียนใหม่จะถูก LLM เทียบกับต้นฉบับในแง่ความหมาย ไม่ใช่แค่ภาษา (เกณฑ์เริ่มต้น: คะแนน ≥ 70 ปรับได้ผ่าน `judgeThreshold` ใน `config.json`) ถ้าความหมายไม่ตรงกัน คุณจะได้แค่คำใบ้ ไม่มีวันได้คำแปลเต็มๆ ดังนั้นคุณต้องคิดเองให้ออก
- **giveup** — พิมพ์ `giveup` ตอนไหนก็ได้ระหว่างที่ถูก block หรือขอให้เขียนใหม่ เพื่อยอมแพ้ออกไป คุณจะเห็นคำแปลภาษาอังกฤษและมันจะถูกส่งผ่านไปเลย ส่วนคู่ประโยคก็ยังถูกบันทึกไว้ให้ทบทวนตามปกติ
- **คิว** — เก็บไว้ที่ `~/.english-by-default/queue.jsonl` หนึ่ง JSON object ต่อหนึ่งบรรทัด จำกัดไว้ที่ 1000 รายการล่าสุดโดยค่าเริ่มต้น (`queueSize` ใน `config.json`)
- **Stopwords (คำผ่านด่วน)** — รายการที่ถูกปล่อยผ่านเพราะตรงกับ stopword จะถูกบันทึกไว้โดยไม่มีการแปลแบบ synchronous (เพื่อไม่ให้เรื่องด่วนจริงๆ ถูกหน่วงเวลา) ส่วนฉบับภาษาอังกฤษจะถูกเติมให้ทีหลังแบบ lazy ในครั้งถัดไปที่คุณทบทวนหรือทำแบบทดสอบรายการนั้น
- **Fail-open** — ไม่ว่าการเรียก LLM จะล้มเหลว หมดเวลา หรือ hook เกิด exception พรอมป์จะถูกปล่อยผ่านโดยไม่แก้ไขอะไรเสมอ เครื่องมือนี้ต้องไม่มีวันเป็นสาเหตุที่ทำให้งานที่ถูกต้องถูกบล็อก
- **ไม่มี dependency ใดๆ (zero dependencies)** ใช้ได้กับ Node.js >= 18

## การทดสอบ

```bash
npm test
```

รันชุด unit test แบบไม่มี dependency (`node --test test/unit.test.js` ครอบคลุม `detect.js`, `queue.js`, `state.js`, `extractJSON` ใน `llm.js`, และ `i18n.js`) ตามด้วย smoke test (`test/smoke.sh`) ซึ่งรันกระบวนการ hook ทั้งหมดแบบ end-to-end จริงกับ LLM stub ปลอม (`test/fake-llm.js`) — ไม่มีการเรียก LLM จริงเกิดขึ้นเลย
