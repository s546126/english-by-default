# English by Default

«Английский шлюз» для ИИ-CLI. Он перехватывает не-английские промпты ещё до того, как они попадут в модель, заставляет вас переписать их на английском (либо сдаться и получить готовый перевод), а из каждой собранной пары со временем формирует материал для тренировок по методу интервальных повторений и технике Фейнмана.

Нативно работает с Claude Code, а с любым другим AI CLI (Codex, Grok, ...) — через небольшую команду-обёртку.

## Как это работает

```
You type: "帮我重构这个函数,保持接口不变"
   │
   ├─ block mode  ──→ 🛡 Intercepted. Please rewrite it in English.
   │                   You: "please refactor this function"
   │                   LLM judge: ❌ score 55 — hint: missing "keep the interface unchanged"
   │                   You: "refactor this function but keep the interface unchanged"
   │                   LLM judge: ✅ score 93 — allowed through, pair logged
   │                   (type giveup at any point to get the English phrasing and continue)
   │
   ├─ warn mode  ──→ ⚠️ Warned, auto-translated to English, and logged
   ├─ log mode   ──→ Silently translated and logged
   └─ hits a stopword (e.g. "urgent"/"紧急") ──→ ⚡ block skipped, allowed straight through, still logged
```

Каждый перехваченный промпт проходит один и тот же конвейер:

1. **Обнаружение (Detect)** — является ли текст неанглийским? (`src/detect.js`; что именно считается «неанглийским», подробно объясняется ниже в разделе [Многоязычный ввод](#многоязычный-ввод).)
2. **Принятие решения (Decide)** — в зависимости от текущего режима (`block` / `warn` / `log`) промпт либо блокируется с просьбой переписать его на английском, либо переводится и пропускается дальше с пометкой.
3. **Оценка (Judge)** — в режиме `block` ваш английский вариант сравнивается с оригиналом с помощью LLM на предмет смысловой эквивалентности (важно не просто «это по-английски», а «значит ли это то же самое»). Если переписанный вариант теряет требование или меняет смысл, вы получаете подсказку и ещё одну попытку, а не готовый перевод — цель в том, чтобы вы сформулировали его сами.
4. **Запись (Log)** — пара «оригинал/английский вариант» добавляется в локальную очередь JSONL для последующего повторения.

Изначально спроектировано по принципу fail-open: если вызов LLM завершился ошибкой, истёк тайм-аут или сам хук выбросил исключение, ввод пропускается без изменений. Этот инструмент никогда не должен становиться причиной, по которой заблокирована ваша реальная работа.

## Установка

### Рекомендуемый способ: плагин для Claude Code

Репозиторий распространяется как нативный плагин для Claude Code (`.claude-plugin/marketplace.json` + `.claude-plugin/plugin.json` + `hooks/hooks.json`), поэтому клонировать репозиторий или выполнять `npm link` не требуется:

```
/plugin marketplace add s546126/english-by-default
/plugin install english-by-default@english-by-default
```

Вот и всё — хук `UserPromptSubmit` подключается автоматически и сразу начинает фильтровать ваши промпты в режиме `block` (используется по умолчанию).

### Ручная установка / автономный CLI

Этот путь всё ещё нужен, если вы хотите использовать команду `ebd` напрямую — `ebd x`, `ebd gate`, `ebd quiz`, `ebd review`, `ebd feynman`, `ebd web` и так далее работают полностью вне Claude Code, а установка плагина, описанная выше, не добавляет бинарник `ebd` в ваш `PATH`.

```bash
git clone https://github.com/s546126/english-by-default.git
cd english-by-default
npm link                  # gives you the global `ebd` command
ebd install claude-code    # writes a UserPromptSubmit hook into ~/.claude/settings.json
```

`ebd uninstall claude-code` отменяет установку хука.

### LLM-бэкенд

По умолчанию перевод и оценка выполняются через локальный CLI `claude -p --model haiku`. Вы можете указать в `llm.command` файла `~/.english-by-default/config.json` любой CLI, где последним аргументом передаётся промпт, а ответ приходит на stdout (например, `codex exec`).

## Использование

```bash
ebd mode block|warn|log     # strength: block | warn | log-and-pass (default: block)
ebd stopwords list|add <w>|rm <w>   # emergency words: matching one skips the block entirely
ebd list [n]                 # show the last n original/English pairs (default 10)
ebd stats                    # queue + review stats

ebd quiz [n]                 # pop quiz: see the original, write the English from memory, LLM scores it (default 5)
ebd review                   # spaced-repetition review: only due entries, pass promotes the interval, fail demotes it
ebd feynman                  # Feynman technique: explain the idea in the simplest English you can, LLM probes for gaps
```

### Другие CLI (Codex / Grok и любые другие)

Для Claude Code используется нативный хук. Все остальные инструменты работают через обёртку, которая фильтрует не-английские аргументы перед вызовом самого инструмента:

```bash
ebd x codex exec "帮我把这个接口改成分页的"
ebd x grok "解释一下这段代码"
ebd gate "帮我写个爬虫"     # non-interactive: prints the English version and logs the pair, good for scripts
```

`ebd x` анализирует только текст, переданный ему *в виде аргументов командной строки*, — он не видит ничего, что вы вводите в интерактивном REPL/TUI после запуска обёрнутого инструмента (такой ввод идёт напрямую в stdin дочернего процесса). Подробности по каждому инструменту см. в [`docs/ADAPTERS.md`](../ADAPTERS.md), включая то, у каких CLI есть собственный нативный хук в духе `UserPromptSubmit`, который можно направить прямо на `hooks/claude-code-hook.js`, не используя обёртку.

## Многоязычный ввод

Определение языка намеренно основано **на диапазонах Unicode-письменностей**, а не является полноценным классификатором языков. `src/detect.js` помечает текст как неанглийский, если в нём встречаются два или более символа из не-латинского диапазона Unicode (CJK, кана, хангыль, кириллица, иврит, арабское письмо, деванагари, тайское письмо). Перед подсчётом из текста вырезаются фрагменты внутри блоков кода и inline-код, поэтому вставка сниппета с не-английскими комментариями или строками не срабатывает как триггер.

Если текст всё же срабатывает как триггер, функция `detectLanguage()` из `src/detect.js` дополнительно уточняет, *какой именно* это язык, опираясь исключительно на задействованную Unicode-письменность, а `src/i18n.js` использует эту информацию, чтобы локализовать сами сообщения блокировки/предупреждения/подсказки (сам промпт в любом случае переводится на английский с помощью LLM — это влияет только на то, на каком языке показываются сообщения самого инструмента). Поддерживаемые языки:

| Код | Язык | Письменность, по которой определяется |
|---|---|---|
| `zh` | китайский | иероглифы CJK при отсутствии каны |
| `ja` | японский | наличие хираганы/катаканы (отличает от `zh`, так как кандзи пересекаются с китайскими иероглифами) |
| `ko` | корейский | хангыль |
| `ru` | русский | кириллица |
| `he` | иврит | еврейское письмо |
| `ar` | арабский | арабское письмо |
| `hi` | хинди | деванагари |
| `th` | тайский | тайское письмо |

**Языки на основе латинского алфавита намеренно не определяются и не блокируются.** Испанский, французский, немецкий, португальский, итальянский, нидерландский и похожие языки используют те же латинские буквы, что и английский, поэтому проверка по диапазону письменности не может отличить их от английского (или от английского заимствования с диакритикой вроде «café»). Чтобы различать их, потребовался бы словарь или статистическая эвристика, а это осознанно выходит за рамки данной функции — это известное и намеренное ограничение, а не баг. Промпты на этих языках сейчас проходят через фильтр без блокировки.

## Веб-панель

```bash
ebd web [port]   # default port 4173
```

Запускает локальную панель только для чтения (обычный модуль Node `http`, без зависимостей), которая показывает:

- Общее число записанных пар, средний балл вспоминания/повторения, сколько записей ожидают повторения, и разбивку по режимам (`rewrite` / `warn` / `log` / `giveup` / `stopword` / ...).
- Часто встречающиеся слова/фразы из ваших сохранённых оригиналов (для текста на латинице — по словам, для текста в духе CJK — по биграммам символов, поскольку без словаря там нет естественных границ между словами).
- Недавние выражения, которые оценщик естественности пометил как **не** идиоматичные (`natural: false`), вместе с более естественной формулировкой, предложенной LLM.

**Замечание по безопасности:** сервер привязывается только к `127.0.0.1` и не имеет никакой аутентификации. Он рассчитан на просмотр с той же машины. Не выставляйте его через публичный порт, туннель или любой обратный прокси, открывающий доступ из сети, — любой, кто сможет до него достучаться, сможет прочитать всю историю ваших промптов.

## Подробности

- **Смысловая оценка (semantic judging)** — в режиме `block` ваш переписанный английский вариант сравнивается LLM с оригиналом по смыслу, а не только по языку (порог по умолчанию: score ≥ 70, настраивается через `judgeThreshold` в `config.json`). При несовпадении вы получаете подсказку, но никогда — готовый перевод, так что додумывать приходится самому.
- **giveup** — введите `giveup` на любом экране блокировки/переписывания, чтобы сдаться: вам покажут английский перевод, он будет пропущен дальше, а пара всё равно попадёт в очередь для повторения.
- **Очередь (Queue)** — хранится в `~/.english-by-default/queue.jsonl`, по одному JSON-объекту на строку, по умолчанию ограничена последними 1000 записями (`queueSize` в `config.json`).
- **Стоп-слова (Stopwords)** — записи, пропущенные благодаря совпадению со стоп-словом, логируются без синхронного перевода (чтобы ничто не замедляло вас в реальной экстренной ситуации); английская версия дозаполняется лениво — при следующем повторении или квизе по этой записи.
- **Fail-open** — любая ошибка вызова LLM, тайм-аут или исключение в хуке пропускают промпт без изменений. Этот инструмент никогда не должен становиться причиной блокировки легитимной работы.
- **Нулевые зависимости**, Node.js >= 18.

## Тесты

```bash
npm test
```

Запускает набор модульных тестов без зависимостей (`node --test test/unit.test.js`, покрывающий `detect.js`, `queue.js`, `state.js`, `extractJSON` из `llm.js` и `i18n.js`), а затем дымовой тест (`test/smoke.sh`), который прогоняет весь конвейер хука целиком на фиктивной заглушке LLM (`test/fake-llm.js`) — никаких реальных вызовов LLM при этом не происходит.
